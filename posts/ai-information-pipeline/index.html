为什么我用AI重构了信息获取方式

2026年4月28日 · 1 分钟 · 龙大

问题的起点#

每天早上醒来，习惯性打开手机：财经新闻、行业动态、技术趋势……信息量大、来源分散、质量参差不齐。

作为一个关注投资和科技的人，我每天需要处理的信息大概分三类：

  • 财经信息 — A股走势、美股盘前、宏观政策
  • 技术动态 — AI前沿、开源项目、硬件评测
  • 行业观察 — 电销获客、内容创业、副业探索

手动刷一遍，至少要花40分钟。而且信息是碎片化的，很难形成决策依据。

方案：AI + 定时推送#

最终的方案是搭建一个自动化信息闭环：

执行细节#

信息源配置

  • Hacker News / GitHub Trending — 技术前沿
  • 36氪 / 腾讯新闻 — 国内商业
  • WallStreetCN — 财经市场
  • 微博热搜 — 舆情风向

推送时段设计

时间内容设计意图
08:30早间新闻速递通勤前快速扫读
09:00AI学习汇报开工前了解AI动态
12:00Polymarket预测市场午休时关注全球事件概率
14:00午间新闻下午工作前补充
21:00晚间新闻下班后深度阅读

质量控制

每条新闻强制标注来源和时间，不做无源信息。筛选逻辑偏重"对决策有用"，而非"标题吸引人"。

关键收益#

  1. 时间节省 — 从每天40分钟降到15分钟碎片阅读
  2. 信息质量 — AI预筛选过滤噪音，留存高密度信息
  3. 决策闭环 — 晨间看市场、午间看趋势、晚间看深度，形成完整的日决策节奏

踩过的坑#

  • 第一版推送太密集，信息焦虑反而加重 → 精简为五个时段
  • 网络波动导致推送中断 → 设计兜底机制，断流时主动补推
  • 定时任务accountId配置错误 → 逐一排查六个任务，已修复

下一步#

  • 加入更多垂直领域信息源（如量化策略回测数据）
  • 与博客联动，定期将深度分析整理成长文
  • 探索基于市场信号的自适应推送频率

信息是投资决策的燃料，但只有经过筛选和结构化的信息才是好燃料。
